您必須向 Google 提出審查要求，如果我們判斷您的網站或網頁沒有危險性或存在欺騙行為後，就會移除相關標記。

您需要：

  • 殼層/終端機指令知識

實際行動：

1. 先決條件

提出審查申請之前，請確認您已完成以下步驟：

  • 在 Search Console 中驗證您的網站擁有權
  • 修復駭客對網站所做的破壞
  • 修正漏洞
  • 將修復的網站重新上線

2. 檢查您的網頁是否可用且無害

為了安全起見，請使用 Wget 或 cURL 查看您網站上的網頁，包括曾遭駭客修改、現已修正問題的首頁和網址。如果有害內容均已清除，而且您也確信網站上的其他網頁也是如此，即可提出審查要求。

重要資訊：您的網頁必須能讓 Googlebot 檢索，才能確保內容無害。請確定您並未使用「noindex」漫遊器中繼標記或指令封鎖 Googlebot 或禁止建立索引。

3. 要求審查

要求審查之前，請確認所有問題均已修復完成；在問題仍未解決的情況下提出審查要求，只會延長網站標記為具有危險性的時間。

  • 網路釣魚
    1. 前往 google.com/safebrowsing/report_error/ 完成審查。只要通過這個審查，確認網頁上已沒有網路釣魚內容，我們就會將網路釣魚警告移除。如果網站擁有者確信自己的網頁遭到錯誤檢舉，也可以前往該網址進行回報。
  • 惡意軟體或垃圾內容
    1. 開啟 Search Console 的安全性問題報告。這份報告可能仍會顯示警告，以及您先前看到的受感染網址的示例。
    2. 如果您確信畫面中列出的示例網址均已清除有害內容，請選取 [要求審查]。為了滿足審查需要，您必須提供進一步的資訊，證明您的網站已徹底清除駭客造成的損害。比方說，對於「安全性問題」中的各個類別，您可以寫幾句話說明自己清理網站的做法 (例如：「對於遭駭客植入內容的網址，我移除了垃圾內容，並且以更新過期外掛程式的方式修正了漏洞」)。

4. 等待審查完成

  • 惡意軟體的審查處理時間：對於遭惡意軟體感染的網站，我們需要幾天的時間進行審查。一旦審查完成，您就會在 Search Console 的「訊息」中收到回應。
  • 垃圾內容的審查處理時間：對於遭入侵放置垃圾內容的網站，，我們可能需要多達幾個星期的時間進行審查。這是因為垃圾內容的審查作業可能需要人工調查，也可能要重新處理遭入侵的網頁。網站通過審查後，「安全性問題」中將不會再顯示入侵的類型，或是任何遭入侵網址的示例。
  • 網路釣魚的審查處理時間：網路釣魚的審查大約需要一天的時間。如果成功通過審查，我們就會將對使用者顯示的網路釣魚警告移除，您的網頁也會再次顯示在搜尋結果中。

一旦 Google 確認您的網站沒有任何問題，就會在 72 小時內將警告從瀏覽器和搜尋結果中移除。

如果 Google 判定您尚未修正問題，Search Console 的「安全性問題」報告將會列出更多受感染網址的示例以協助您之後的調查。同時，為了保護使用者的安全，搜尋結果和/或瀏覽器中仍會顯示惡意軟體或網站遭到垃圾內容入侵的警告。

最後步驟

  • 如果您的網站通過審查，請確認網站是否如預期運作，是否能順利載入網頁，有沒有不能點選的連結。為了維護您的網站安全，我們鼓勵所有網站擁有者一起來實作清理及維護您的網站中建立的維護和安全性計劃。如需更多資訊，請參閱下列來自 StopBadware 的資源：
    • 防堵惡意程式：基本概念
    • 其他資源：網站遭入侵
  • 如果您的網站未獲得核准，請再次檢查您的網站是否有惡意軟體、垃圾內容，或者駭客竄改或新建立的檔案。此外，您也可以向支援小組的專家尋求更多協助。
