คุณต้องขอให้ Google ตรวจสอบเพื่อให้มีการยกเลิกการตั้งค่าสถานะหน้าเว็บหรือไซต์ว่าเป็นอันตรายหรืออาจหลอกลวงผู้ใช้

คุณจะต้องมีสิ่งต่อไปนี้

  • ความรู้เกี่ยวกับคำสั่ง Shell/เทอร์มินัล

สิ่งที่คุณต้องดำเนินการ

1. ข้อกำหนดเบื้องต้น

ก่อนขอให้ตรวจสอบ ให้ตรวจดูว่าคุณได้ทำตามขั้นตอนต่อไปนี้แล้ว

  • ยืนยันการเป็นเจ้าของไซต์ใน Search Console
  • ทำความสะอาดสิ่งที่แฮ็กเกอร์ได้ทำให้เสียหายในไซต์
  • แก้ไขช่องโหว่
  • นำไซต์ปกติกลับมาออนไลน์

2. ตรวจสอบซ้ำอีกครั้งว่าหน้าเว็บใช้งานได้และมีข้อมูลที่เป็นปกติ

เพื่อความปลอดภัย ให้ใช้ Wget หรือ cURL เพื่อดูหน้าต่างๆ ในไซต์ เช่น หน้าแรกและ URL ที่โดนแฮ็กเกอร์แก้ไข ซึ่งตอนนี้ควรจะแสดงข้อมูลเป็นปกติแล้ว หากหน้าดังกล่าวไม่มีปัญหาใดๆ และคุณมั่นใจว่าหน้าอื่นๆ ในไซต์ก็เป็นปกติแล้วเช่นกัน คุณก็ส่งคำขอให้ตรวจสอบได้

สำคัญ: หน้าเว็บต้องพร้อมให้ Googlebot รวบรวมข้อมูลเพื่อให้แน่ใจว่าปลอดจากอันตรายแล้วจริง โดยตรวจสอบว่าไม่ได้มีการป้องกันโดยใช้ robots หรือบล็อกไม่ให้จัดทำดัชนีโดยใช้เมตาแท็กหรือคำสั่ง "noindex" ของ robots

3. ขอให้ตรวจสอบ

ก่อนขอให้ตรวจสอบ ให้ตรวจดูว่ามีการแก้ไขปัญหาต่างๆ แล้วจริง ถ้าขอให้ตรวจสอบโดยที่ยังมีปัญหาอยู่ ก็มีแต่จะทำให้ไซต์มีสถานะอันตรายเนิ่นนานต่อไปอีก

  • ฟิชชิง
    1. กรอกฟอร์มเพื่อขอให้ตรวจสอบที่ google.com/safebrowsing/report_error/ การตรวจสอบนี้จะนำคำเตือนฟิชชิงออกจากหน้าฟิชชิงที่ทำความสะอาดแล้ว และยังใช้เป็นเครื่องมือการรายงานสำหรับเจ้าของไซต์ที่เชื่อว่าหน้าเว็บถูกตั้งค่าสถานะอย่างไม่ถูกต้องว่ามีการฟิชชิง
  • มัลแวร์หรือสแปม
    1. เปิดรายงานปัญหาด้านความปลอดภัยใน Search Console รายงานอาจจะยังแสดงคำเตือนและตัวอย่าง URL ที่ติดเชื้อซึ่งคุณเคยเห็นมาก่อนแล้ว
    2. หากคุณเชื่อว่าตัวอย่าง URL ที่ระบุนั้นเป็นปกติทั้งหมดแล้ว ให้เลือกขอให้ตรวจสอบ ในการส่งคำขอให้ตรวจสอบ โปรดให้ข้อมูลเพิ่มเติมเกี่ยวกับการจัดการความเสียหายในไซต์ที่เกิดจากแฮ็กเกอร์ให้เป็นปกติ ตัวอย่างเช่น สำหรับแต่ละหมวดหมู่ใน "ปัญหาด้านความปลอดภัย" คุณสามารถเขียนข้อความที่อธิบายว่า ไซต์ได้รับการแก้ไขอย่างไร (เช่น "สำหรับ URL ที่ถูกแฮ็กโดยการแทรกเนื้อหา ฉันได้ลบเนื้อหาสแปมออก และแก้ไขช่องโหว่แล้วด้วยการอัปเดตปลั๊กอินที่ล้าสมัย")

4. รอให้มีการตรวจสอบ

  • ระยะเวลาในการตรวจสอบมัลแวร์: การตรวจสอบไซต์ที่ติดมัลแวร์จะใช้เวลาประมาณ 2-3 วัน หลังจากที่การตรวจสอบเสร็จสิ้น คุณจะดูการตอบกลับได้ในข้อความใน Search Console
  • ระยะเวลาในการตรวจสอบการถูกแฮ็กให้แสดงสแปม: การตรวจสอบไซต์ที่ถูกแฮ็กให้แสดงสแปมอาจใช้เวลาหลายสัปดาห์ เนื่องจากอาจต้องมีการตรวจสอบสแปมโดยเจ้าหน้าที่หรือตรวจสอบหน้าที่ถูกแฮ็กใหม่ทั้งหมด ถ้าผลการตรวจสอบออกมาว่าผ่าน "ปัญหาด้านความปลอดภัย" จะไม่แสดงหมวดหมู่ที่ถูกแฮ็กหรือตัวอย่าง URL ที่ถูกแฮ็กอีกต่อไป
  • ระยะเวลาในการตรวจสอบฟิชชิง: การตรวจสอบฟิชชิงจะใช้เวลาประมาณ 1 วัน หากผลการตรวจสอบออกมาว่าผ่าน คำเตือนฟิชชิงที่ปรากฏต่อผู้ใช้จะถูกนำออกและหน้าเว็บอาจปรากฏในผลการค้นหาอีกครั้ง

หาก Google พบว่าไซต์ทำงานเป็นปกติแล้ว จะมีการนำคำเตือนในเบราว์เซอร์และผลการค้นหาออกภายใน 72 ชั่วโมง

หาก Google ตัดสินว่ายังคงมีปัญหาอยู่ ปกติแล้วรายงาน "ปัญหาด้านความปลอดภัย" ใน Search Console จะแสดงตัวอย่าง URL ที่ติดเชื้อเพิ่มเติมเพื่อช่วยคุณในการตรวจสอบต่อไป คำเตือนว่าไซต์มีมัลแวร์หรือถูกแฮ็กให้แสดงสแปมจะยังคงปรากฏในผลการค้นหาและ/หรือเบราว์เซอร์เพื่อเป็นการเตือนผู้ใช้ให้ระมัดระวัง

ขั้นตอนสุดท้าย

  • หากคำขอผ่านการอนุมัติ ให้ตรวจสอบว่าไซต์ทำงานตามที่ควรจะเป็น เช่น หน้าเว็บโหลดได้เป็นปกติและสามารถคลิกลิงก์ได้ ในการดูแลไซต์ให้ปลอดภัย เจ้าของไซต์ทุกรายควรใช้แผนการดูแลรักษาและการรักษาความปลอดภัยที่สร้างขึ้นในขั้นตอนการทำความสะอาดและดูแลรักษาไซต์ สำหรับข้อมูลเพิ่มเติม โปรดดูทรัพยากรจาก StopBadware ดังต่อไปนี้
    • การป้องกันแบดแวร์: ข้อมูลเบื้องต้น
    • ทรัพยากรเพิ่มเติม: ไซต์ที่ถูกแฮ็ก
  • หากคำขอไม่ผ่านการอนุมัติ ให้ประเมินไซต์อีกครัั้งเพื่อหามัลแวร์หรือสแปม หรือหาการแก้ไขไฟล์หรือไฟล์ใหม่ๆ ที่แฮ็กเกอร์สร้างขึ้น หรือจะลองขอความช่วยเหลือเพิ่มเติมจากผู้เชี่ยวชาญในทีมสนับสนุนก็ได้
