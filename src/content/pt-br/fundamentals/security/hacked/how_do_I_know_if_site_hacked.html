Siga estas etapas se:

  • você foi alertado pelo Google sobre a invasão do seu site;
  • você vir um aviso na pesquisa informando que o site foi invadido ou comprometido;
  • você não tiver certeza se o site foi realmente invadido.

Se você não tiver certeza de que o site foi realmente invadido ou achar que o site foi marcado incorretamente, o primeiro passo é o registro do site no Search Console. Acesse as seções Ações manuais ou Problemas de segurança do Search Console e procure os URLs de exemplo em que o Google detectou que o site foi invadido.

Se você não conseguir ver conteúdo de invasão nos URLs fornecidos no Search Console, pode ser que esteja sendo usada uma técnica conhecida como cloaking. As técnicas de cloaking tornam a limpeza do site mais difícil, porque conteúdos diversos são exibidos para diferentes tipos de usuários. Por exemplo, quando você acessar uma página no site, como www.example.com/cheap-drugs, verá uma página sem conteúdo. Isso pode levar você a acreditar que não existe conteúdo de invasão no site. No entanto, quando um mecanismo de pesquisa como o Google acessar www.example.com/cheap-drugs, ele verá palavras e links com spam.

Para verificar se há técnicas de cloaking, use o Solucionador de problemas para sites invadidos. O solucionador de problemas orientará você sobre algumas ferramentas, como o operador de pesquisa site: e o Fetch as Google, que podem ajudá-lo a descobrir a presença de conteúdos de invasão.

Depois de verificar com cuidado suas páginas, se você ainda achar que o site foi marcado incorretamente, faça uma postagem nos Fóruns de Ajuda para webmasters. Caso contrário, siga o guia de ajuda dos webmasters para sites invadidos para limpar seu site.
