globDirectory

Required String

The base directory you wish to match globPatterns against, relative to the current working directory.

Example: 

// Treat all patterns as relative to the current directory:
globDirectory: '.'

importScripts

Optional Array of String, defaulting to []

An optional list of JavaScript files that should be passed to importScripts() inside the generated service worker file.

This is useful when you want to let Workbox create your top-level service worker file, but also want to include some additional code, such as a push event listener.

Note: It's not necessary to include a URL for the workbox-sw.js runtime code here, as it will automatically be added.

Example: 

importScripts: ['push-listener.abcd1234.js']

importWorkboxFromCDN

Optional Boolean, defaulting to true

If true, the workbox-sw.js runtime and associated libraries will be automatically imported into the generated service worker from the official CDN URL.

If false, the workbox-sw.js runtime and associated libraries will be copied locally into a versioned subdirectory of swDest.

Functionality is identical regardless of which option is used; setting this to false is recommended for developers who would prefer not to rely on the official CDN as a dependency.

Example: 

importWorkboxFromCDN: false

swDest

Required String

The path to the final service worker file that will be created by the build process, relative to the current working directory.

Example: 

swDest: path.join('dist', 'sw.js')
