{# wf_updated_on: 2018-05-21 #} Chrome will automatically show the Add to Home Screen prompt to the user when the following criteria are met:
  • The web app is not already installed
  • Meets a user engagement heuristic (currently, the user has interacted with the domain for at least 30 seconds)
  • Meets the Progressive Web App criteria:
    • Includes a web app manifest that includes:
      • short_name or name
      • icons must include a 192px and a 512px sized icons
      • start_url
      • display must be one of: fullscreen, standalone, or minimal-ui
    • Served over HTTPS (required for service workers)
    • Has registered a service worker with a fetch event handler
If the web app manifest includes related_applications and has "prefer_related_applications": true, the native app install prompt will be shown instead.
Caution: Beginning in Chrome 68 (May 2018), Chrome will not automatically show the Add to Home Screen prompt, instead, you must trigger it by using the beforeinstallprompt event.
