위험하거나 사기 가능성이 있는 페이지 또는 사이트로 표시된 경우 표시를 해제하려면 Google에 검토를 요청해야 합니다.

필요 사항:

  • 셸/터미널 명령에 대한 지식

수행할 작업:

1. 선행 조건

검토를 요청하기 전에 다음 단계를 수행했는지 확인합니다.

  • Search Console에서 사이트 소유권 확인
  • 해커로부터 피해를 입은 사이트 치료
  • 취약성 해결
  • 사이트를 치료하여 다시 온라인으로 전환

2. 페이지를 다시 사용할 수 있고 안전한지 다시 확인

안전을 위해, 해커가 수정한 홈페이지 및 URL 등 치료되었다고 생각되는 사이트의 페이지를 Wget 또는 cURL을 사용하여 표시합니다. 확인 후 사이트의 나머지 페이지도 안전하다고 확신하는 경우 검토를 요청하시기 바랍니다.

중요: 페이지가 안전한지 확인하려면 Googlebot이 페이지를 크롤링할 수 있어야 합니다. 페이지가 'noindex' robots 메타태그나 명령어로 로봇에 의해 차단되거나 색인 생성이 차단되어 있으면 안 됩니다.

3. 검토 요청

검토를 요청하기 전에 문제가 확실히 해결되었는지 확인합니다. 문제가 해결되지 않았는데 검토를 요청하면 안전하지 않은 사이트로 표시되는 기간만 더 길어질 뿐입니다.

  • 피싱
    1. google.com/safebrowsing/report_error/ 페이지에서 검토를 완료합니다. 이 검토는 정리된 피싱 페이지에서 피싱 경고를 삭제합니다. 또한 페이지가 피싱으로 잘못 신고되었다고 생각하는 사이트 소유자를 위한 보고 도구로 제공됩니다.
  • 멀웨어 또는 스팸
    1. Search Console에서 보안 문제 보고서를 엽니다. 보고서에는 이전에 표시되었던 경고 및 감염된 샘플 URL이 계속 표시될 수 있습니다.
    2. 표시된 샘플 URL이 모두 안전하다고 판단되면 검토 요청을 선택합니다. 검토를 제출하려면 사이트에서 해커로 인한 피해가 제거되었다는 자세한 정보를 Google에 제공해야 합니다. 예를 들어 보안 문제 내 각 카테고리의 경우 사이트의 문제를 해결한 방법을 설명하는 문장을 작성해야 합니다(예: '콘텐츠 삽입으로 해킹된 URL의 경우 스팸성 콘텐츠를 삭제하고 오래된 플러그인을 업데이트하여 취약성을 해결했습니다.').

4. 검토 완료까지 대기

  • 멀웨어 검토 처리 시간: 멀웨어에 감염된 사이트의 검토를 처리하려면 며칠이 필요합니다. 검토가 완료되는 대로 Search Console의 메시지로 응답이 발송됩니다.
  • 스팸으로 해킹된 사이트 검토 처리 시간: 스팸으로 해킹된 사이트의 검토를 처리하는 데에는 최대 몇 주가 걸릴 수 있습니다. 스팸을 검토하려면 수동으로 조사하거나 해킹된 페이지를 완전히 다시 처리해야 할 수 있기 때문입니다. 검토가 승인되면 보안 문제에 더 이상 해킹 카테고리 유형 또는 해킹된 URL 예가 표시되지 않습니다.
  • 피싱 검토 처리 시간: 피싱 검토는 처리하는 데 하루 정도 걸립니다. 검토를 통해 승인되면 사용자에게 표시되는 피싱 경고가 삭제되고 페이지가 다시 검색결과에 표시될 수 있습니다.

Google에서 사이트가 안전하다고 판단하면 72시간 이내에 브라우저 및 검색결과에서 경고가 삭제됩니다.

Google에서 문제가 해결되지 않은 것으로 판단하면 다음에 조사하는 데 도움이 되도록 Search Console의 보안 문제 보고서에 감염된 샘플 URL이 추가로 표시됩니다. 멀웨어에 노출되었거나 스팸으로 해킹된 사이트 경고는 사용자 보호를 위한 예방조치로서 계속 검색결과 또는 브라우저에 남아 있게 됩니다.

마지막 단계

  • 요청이 승인된 경우 사이트가 정상적으로 작동하는지, 즉 페이지가 올바로 로드되고 링크가 클릭 가능한지 확인합니다. 사이트를 안전하게 유지하려면 모든 사이트 소유자가 사이트 정리 및 유지관리에 작성된 유지관리 및 보안 계획을 구현하는 것이 좋습니다. 자세한 내용은 StopBadware의 다음 리소스를 참고하세요.
    • 배드웨어 방지: 기본사항
    • 추가 리소스: 해킹된 사이트
  • 요청이 승인되지 않은 경우 사이트에 멀웨어 또는 스팸이 있거나 해커에 의한 수정사항 또는 새로 작성된 파일이 있는지 다시 검토합니다. 또는 지원팀의 전문가에게 도움을 요청할 수도 있습니다.
