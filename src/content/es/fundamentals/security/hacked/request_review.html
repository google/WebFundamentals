Para que tu página o sitio web dejen de estar marcados como peligrosos o como posiblemente engañosos para los usuarios, debes solicitar una revisión a Google.

Lo que necesitarás:

  • Conocimientos de los comandos de shell/terminal

Lo que debes hacer:

1. Requisitos previos

Antes de solicitar una revisión, asegúrate de que hayas hecho lo siguiente:

  • Has demostrado que eres el propietario de tu sitio web en Search Console.
  • Has eliminado del sitio el contenido malicioso de los hackers.
  • Has corregido la vulnerabilidad.
  • Has vuelto a conectar el sitio web.

2. Comprueba que las páginas estén disponibles y limpias

Para mayor seguridad, utiliza Wget o cURL para ver las páginas del sitio web, por ejemplo, la página principal y una URL modificada por el hacker que ahora tendrían que estar limpias. Si es el caso y has comprobado que el resto de las páginas del sitio web también estén limpias, ya puedes solicitar una revisión.

Importante: Para verificar que las páginas estén limpias, Googlebot debe poder rastrearlas. Comprueba que no estén excluidas en el archivo robots.txt y que no se haya bloqueado su indexación por haberse añadido "noindex" a las metaetiquetas o directivas de los robots.

3. Solicita una revisión

Antes de solicitar una revisión, asegúrate de que el problema se haya corregido de verdad. Si solicitas una revisión y el problema sigue ahí, solo lograrás prolongar el tiempo durante el que el sitio web estará marcado como peligroso.

  • Phishing
    1. Completa la revisión que encontrarás en google.com/safebrowsing/report_error/. Con esta revisión se retiran las advertencias de phishing de páginas que han contenido esta amenaza pero que ya están limpias. También sirve como herramienta de información para los propietarios de sitios web que creen que su página se ha marcado incorrectamente como un lugar de phishing.
  • Software malicioso o spam
    1. En Search Console, abre el informe Problemas de seguridad. Es probable que se sigan mostrando las advertencias y URL infectadas de ejemplo que aparecían antes.
    2. Si crees que las URL de ejemplo que se muestran están limpias, selecciona Solicitar una revisión. Para presentar una revisión, debes proporcionarnos información que demuestre que los daños del hacker se han limpiado del sitio. Por ejemplo, para cada categoría de Problemas de seguridad, se puede escribir una frase que explique cómo se limpió el sitio web (por ejemplo, "En cuanto a las URL pirateadas con inyección de contenido, he retirado el contenido fraudulento y he corregido la vulnerabilidad actualizando un complemento desactualizado").

4. Espera a que finalice la revisión

  • Tiempo de procesamiento de la revisión por software malicioso: son necesarios unos cuantos días para procesar las revisiones de sitios web infectados con software malicioso. Cuando la revisión haya finalizado, podrás consultar el resultado en la sección Mensajes de Search Console.
  • Tiempo de procesamiento de la revisión de un sitio web pirateado con spam: se requieren unas cuantas semanas para procesar las revisiones de sitios web pirateados con spam. Esto se debe a que las revisiones de spam pueden implicar una investigación manual o tener que volver a procesar completamente las páginas pirateadas. Si la revisión tiene un resultado positivo, los tipos de la categoría pirateada o los ejemplos de URL pirateadas dejarán de mostrarse en la sección Problemas de seguridad.
  • Tiempo de procesamiento de la revisión por phishing: este tipo de revisiones acostumbran a procesarse en un día aproximadamente. Si el resultado de la revisión es positivo, se retirará la advertencia de phishing que puede ver el usuario y la página se podrá volver a mostrar en los resultados de búsqueda.

Si Google decide que tu sitio web está limpio, se retirarán las advertencias de los navegadores y de los resultados de búsqueda en un plazo de 72 horas.

Si Google determina que no se ha resuelto el problema, en el informe Problemas de seguridad de Search Console se mostrarán más URL infectadas de ejemplo para ayudarte a investigar la causa. Las advertencias de software malicioso o de sitio web pirateado permanecerán en los resultados de búsqueda o en los navegadores como precaución para proteger a los usuarios.

Últimos pasos

  • Si la solicitud ha tenido un resultado positivo, comprueba que el sitio web funcione correctamente, que las páginas se carguen sin problemas y que se pueda hacer clic en los enlaces. Para mantener la seguridad del sitio web, recomendamos a todos los propietarios de sitios web que implementen el plan de mantenimiento y seguridad creado en el paso Limpiar el sitio web y hacer tareas de mantenimiento. Para obtener más información, consulta los siguientes recursos de StopBadware:
    • Prevención de software malicioso: conceptos básicos
    • Recursos adicionales: sitios web pirateados
  • Si la solicitud no se ha aprobado, vuelve a revisar el sitio web para detectar software malicioso o spam, o para comprobar si el hacker ha modificado o creado archivos. También puedes solicitar ayuda a los especialistas del equipo de asistencia.
